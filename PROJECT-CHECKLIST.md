## CHECKLIST MANAGEMENT RULES

### Status Indicators
- ✅ **Done** — Completed, tested, confirmed working
- 🔄 **In Progress** — Currently being worked on (max 2 items at a time)
- ⏳ **Pending** — Queued, waiting for dependencies or resources
- 🚫 **Blocked** — Can't proceed; reason listed in parentheses
- 📌 **On Hold** — Deprioritized or waiting for decision

### Priority Levels
- **CRITICAL** 🔴 — Must be done before launch (blocking everything)
- **HIGH** 🟠 — Should be done this week (important for launch)
- **MEDIUM** 🟡 — Nice to have before launch (can defer 1-2 weeks)
- **LOW** 🟢 — Post-launch or stretch goal (not blocking anything)

### Update Rules
1. **Update this file every session** — After each work block, mark items complete or update status
2. **Move items up if dependencies change** — If an item is no longer blocking, prioritize it
3. **One item per line** — Keep format consistent for easy scanning
4. **Reason for blocked items** — Always explain why something is blocked
5. **This file covers the website only** — All growth/marketing tasks live in GROWTH-CHECKLIST.md

⚠️ **GAINS CONSISTENCY REMINDER:** Avant toute modification de copy sur le site ou dans du contenu commercial, vérifier l'alignement avec [GAINS-REFERENCE.md](../07-sales/GAINS-REFERENCE.md) — baseline: **1+ heure/jour par personne**.
